What is Deadhand Protocol?

Deadhand Protocol is a non-custodial, open-source digital inheritance system that uses Shamir's Secret Sharing to split cryptographic keys into 3 shards.

  • Any 2 shards = full key recovery
  • 1 shard alone = mathematically useless
  • Dead man's switch = if user goes silent for 90 days, shard C is automatically released to beneficiary

Is Deadhand open source?

Yes. Deadhand is fully open source under the MIT license.

  • Source code: github.com/pyoneerc/deadhand
  • Users can self-host their own instance
  • No vendor lock-in: recovery works with standard ssss-split tools

Does Deadhand require KYC?

No. Deadhand does not require identity verification. Users can:

  • Pay with credit card (Stripe) or cryptocurrency
  • Use a pseudonymous email address
  • Self-host for maximum privacy

What secrets can Deadhand protect?

Deadhand works with any secret that can be represented as a seed phrase or recovery key:

How does the dead man's switch work?

  1. User splits secret into 3 shards (A, B, C)
  2. User keeps Shard A
  3. Beneficiary receives Shard B
  4. Shard C is encrypted and stored on Deadhand servers
  5. Every 30 days, Deadhand emails the user to check in
  6. If no response for 90 days, Shard C is automatically released to beneficiary
  7. Beneficiary combines B + C to recover the original secret

What happens if Deadhand shuts down?

Users always retain Shards A and B. These 2 shards are sufficient to recover the original secret using any standard Shamir Secret Sharing tool (e.g., ssss-split on Linux).

The emergency exit is always open.
